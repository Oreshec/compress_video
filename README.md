# Video Compressor

Программа сжимает видео до заданного размера (по умолчанию 10 МБ), предварительно оценивает размер после сжатия и при необходимости снижает битрейт (качество), чтобы уложиться в лимит.

## Требования

- **Python 3.8+**
- **FFmpeg** — должен быть установлен и доступен в PATH ([скачать FFmpeg](https://ffmpeg.org/download.html))

## Использование (консоль)

```bash
# Сжать до 10 МБ (по умолчанию), выход: video_compressed.mp4
python compress_video.py video.mp4

# Задать целевой размер в МБ (например 5 МБ)
python compress_video.py video.mp4 -s 5

# Указать выходной файл
python compress_video.py video.mp4 -o result.mp4 -s 10
```

## Использование (GUI)

```bash
python gui_compressor.py
```

Откроется окно, где можно:

- **выбрать входной файл**;
- **выбрать путь и имя выходного файла**;
- **задать целевой размер в МБ** (по умолчанию 10);
- нажать кнопку **«Сжать»** и дождаться завершения.

### Параметры

| Параметр | Описание |
|----------|----------|
| `input` | Входной видеофайл |
| `-o`, `--output` | Выходной файл (по умолчанию: `имя_входа_compressed.расширение`) |
| `-s`, `--size` | Целевой размер в МБ (по умолчанию: 10) |

## Как это работает

1. **Анализ** — через `ffprobe` определяется длительность, аудио- и видеобитрейт.
2. **Расчёт битрейта** — по целевой длительности и размеру вычисляется нужный видеобитрейт (аудио учитывается отдельно).
3. **Оценка размера** — перед кодированием выводится предварительная оценка размера файла.
4. **Коррекция** — если оценка превышает целевой размер, битрейт уменьшается, пока оценка не уложится в лимит.
5. **Сжатие** — FFmpeg кодирует видео в H.264 и аудио в AAC с рассчитанным битрейтом.

Итоговый размер может немного отличаться от цели из-за особенностей кодека; используется запас ~92%, чтобы чаще укладываться в лимит.
